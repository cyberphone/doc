org.webpki.xml

Class DOMWriterHelper

  • java.lang.Object
    • org.webpki.xml.DOMWriterHelper

  • public class DOMWriterHelper
extends Object
    Utility class making creation of DOM documents easier in simple cases.

    The DOMWriterHelper holds a cursor, the "current element", pointing at the last element created, to simplify building substructures and adding attributes.

    Note that all addXXXX methods, and getParent, do not necessarily act on the "current element" but on the last non-text element created or visited. This is a consequence of the simplified model we use in this class were elements either contain text or subelements (not both). This distinction makes addXXXX methods irrelevant when we are at a text element, instead we then let it act on that text element's parent allowing code like 

       helper.addChildElement("address");
   helper.addString("street", "1 X-Obi Drive");
   helper.addString("city", "Palo Alto");
   helper.addString("state", "CA");
    to create an element address with three subelements. If addTextElement had always operated on the "current element", we would have had to insert getParent between the addTextElements. As an additional consequence of this design, it wouldn't make sense to let getParent operate on text elements. Hence getParent will move the cursor to the parent's parent if the "current element" is a text element.

    A more technical explanation of this would be that there are in fact two cursors, one "element insertion cursor" and one "attribute setting cursor", and that setXXXX methods operate on the element pointed out by the "attribute setting cursor" while the navigation (getParent) and element creation (addXXXX) methods operate on the "element insertion cursor". Then all addXXXX methods and getParent move the "attribute setting cursor" while only getParent, addChildElement and addWrapped also move the "element insertion cursor" (the "attribute setting cursor" will then point to the same element as the "element insertion cursor").

    A small example

       DOMWriterHelper helper = new DOMWriterHelper(doc, "example:order");

   helper.setNamespace("http://www.example.com/xml-schemas/order");

   helper.addString("buyer", buyer);

   helper.setIntAttribute("duns", buyerDUNS);

   if(comment != null)
     helper.addString("comment", comment);
   ...
    which creates the structure (comment!=null in this case) 
       <example:order xmlns:example="...">
       <buyer duns="12345">John Doe</buyer>
       <comment>Urgent!</comment>
       ...
   </example:order>
    This example code comes from an example included in the distribution to illustrate how to write an XMLObjectWrapper.

      • addMoney

        public void addMoney(String name,
                     long value,
                     int numberOfDecimals)
        Add a text element to the current element.

        The text content of the new element will be value divided by 10000 and rounded/extended to the desired number of decimals. The rounding used is BigDecimal.ROUND_HALF_UP, i.e. the standard method of rounding.

        This datatype corresponds to MS SQL Server's money (and smallmoney) datatypes.

        Parameters:
        name - The name of the new element.
        value - The text content of the new element multiplied by 10000.
        numberOfDecimals - The number of decimals to be used the text representation.
        See Also:
        DOMWriterHelper cursor state
